    JSON Database Explained

    For my website I relied on a JSON (JavaScript Object Notation) formatted database. Using this database format I was able to store all the course content within it including references to images and video. JSON is not limited JavaScript development only but definitely helps with development.

    If you aren't familiar with the JSON style of datbase I've included samples.

    The codeblock below provides a basic overview of how JSON works. The root of JSON is wrapped in curly brackes. These curly brackets define the entire database as an object. Within the object we nest key:value pairs. Using JavaScript I can call the key and return the value.

    { "key" : "value" }

    JSON isn't limited to only key:value string pairs. It can also store numbers, booleans of true and false, arrays, and additional objects.

    { "users": [ "uid_1012": { "name": "john", "email": "john@gmail.com", "age": 24, "premier": false, }, ] }

    In the example above, we've included an array [] as key and even nested an additional object {} as a key for the user "uid_1012". If you not a programmer this probably looks confusing. So let's break it down.

    1. From the top we have the entire json object wrapped in the curly brackets. {} - you can see these at the top and bottom.
    2. Directly within the curly brackets we have the keyword: "users". This references an array of users. We've only listed one user: "uid_1012"
    3. If you're not familiar with arrays, we'll keep it simple. It's basically a list.
    4. Now let's focus on the user key of "uid_1012". The value for this key is a new object with multiple key:value pairs separated by a comma.
